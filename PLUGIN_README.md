# 回复救急插件

这是一个 SillyTavern 前端扩展，用来救急已经生成出来的助手回复。新版界面只保留两个入口：

- 重写选中片段：在助手回复正文里选中一段文字，点选区旁或消息按钮区的小魔法棒，只替换选中的这段。
- 自动识别块：点底部魔法棒菜单里的“回复救急插件”打开 UI，自动列出当前聊天最后一条助手回复里正文之外的结构块，可点选一个或多个块后一次修复。

所有改动都必须先生成预览，再手动应用到当前回复。

## 当前版本亮点

- 多选结构块：一次选择多个识别块后统一生成预览，再分别替换原块。
- 缺失块修复：插件会记录当前聊天的块顺序，某轮少块时可显示缺失并按对应位置插回。
- 中途新增块：当某轮回复完整包含旧块并新增结构块时，会自动扩展当前聊天的块基线。
- 删除选中块：可删除当前回复里真实存在的选中块，缺失占位不会误删。
- 大上下文：上下文字符数最高 100000，修复输出长度最高 20000。
- 独立 API：可在当前酒馆模型和独立 API 之间切换；独立 API 支持保存多个预设，并自动识别可用模型。
- 移动端适配：手机端单列布局，块列表、扫描按钮和底部按钮都能自适应换行。

## 修改结构块

1. 点聊天输入框旁边的底部魔法棒菜单。
2. 点“回复救急插件”打开 UI。
3. UI 会自动扫描当前聊天最后一条助手回复；也可以点“扫描当前聊天块”重新读取。
4. 点你要修的块；需要同时修多个块时，连续点选多个。
5. 在“你的修改要求”里写清楚要改什么，例如“把体力改成 60，其他字段不动”或“重新生成这个块，但保留原来的字段顺序和样式”。多选时，同一条要求会分别用于每个选中的块。
6. 点“生成修复预览”，确认无误后点“应用到当前回复”。

消息旁的小魔法棒只负责选中正文片段的重写；没有选中正文时不会进入块修复。

识别不是只认 `<StatusBlock>...</StatusBlock>`。插件会尝试识别：

- XML/类 XML 成对标签，例如 `<StatusBlock>...</StatusBlock>`、`<InventoryPanel>...</InventoryPanel>`。
- 方括号/中文括号成对块，例如 `[状态栏]...[/状态栏]`、`【选项栏】...【/选项栏】`。
- 带 `class`、`id`、`style`、`data-*`、`aria-*` 的 HTML 美化容器，例如 `<div class="status-panel">...</div>`。
- 当前预设、正则和相关世界书里的格式线索，但它们只作为格式证据。

如果某个块在原始消息里完全没有清晰边界，插件不会硬猜。这样能避免把正文误识别成结构块。

## 独立 API 设置

默认生成来源是“当前酒馆模型”。如果想让插件单独使用另一个接口：

1. 打开扩展设置里的“回复救急插件”。
2. 将“生成来源”改为“独立 API”。
3. 填 API 预设名称、API URL、API Key。
4. 点击“识别可用模型”。
5. 选择模型后点击“保存 API 预设”。

独立 API 走 OpenAI 兼容格式：`GET /models` 获取模型列表，`POST /chat/completions` 生成修复内容。API Key 保存在本地 SillyTavern 扩展设置里，只适合个人电脑使用。

## 不胡编的依据规则

生成时只允许使用这些依据：

- 当前选中的旧块或坏块；多选时会分别读取每个选中块。
- 当前助手回复里的正文参考。
- 最近上下文。
- 你在“修改要求”里明确写出的信息。
- 预设、正则、世界书里的格式信息。

预设、正则和世界书只用于确认外层标签、HTML/class、字段顺序、按钮宏和排版，不能当作当前剧情事实。无法确定的字段会要求模型保持旧值；没有旧值时写“未知”或“无”。

新版默认关闭旧版本的“无旧块追加”能力。也就是说，没有当前消息里真实存在的目标块时，不会凭空生成一个新结构块。

## 不会粘住旧对话

每次打开插件 UI 或点击“扫描当前聊天块”都会重新读取当前聊天最后一条助手回复。

切换或加载聊天时，插件会清空当前块识别状态，避免把上一轮对话的块列表带到新对话里。

## 手机和电脑

电脑端在插件 UI 里是左侧块列表、右侧编辑区。

手机端会自动变成单列：块列表在上，原文、修改要求和预览在下，按钮横向排字并按空间换行。
