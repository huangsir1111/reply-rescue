# 回复救急插件

SillyTavern 前端扩展，用来修改已经生成出来的助手回复。当前界面只保留两个入口：

- 重写选中片段：选中助手回复中的一段正文，点小魔法棒，只替换这段文字。
- 自动识别块：点底部魔法棒菜单里的“回复救急插件”，打开 UI 后扫描当前聊天最后一条助手回复，可点选一个或多个块后一次修复。

必须先生成预览，再手动应用。

## 安装

在 SillyTavern 扩展安装界面导入仓库地址：

```text
https://github.com/huangsir1111/reply-rescue
```

## 最新版本亮点

- 支持多选结构块，一次修复多个代码块/状态块/选项块。
- 支持聊天块基线：第一轮结构完整时会记录块的顺序和位置，后续某轮少块时会标记缺失，并按正确位置插回。
- 支持删除选中的已存在结构块；缺失占位不会被误删。
- 上下文字符数最高 100000，修复输出长度最高 20000。
- 支持“当前酒馆模型 / 独立 API”切换。
- 独立 API 可保存多个预设，填写 URL、Key、自定义名称后可通过 SillyTavern 后端识别可用模型并选择保存。
- 独立 API 会在发送前自动整理为更兼容的请求：单条 user 消息、按输入大小压缩格式/风格线索，并自动收敛实际输出上限，减少接口直接拒绝请求。
- 独立 API 支持自动重试次数设置，空输出、格式不合格、限流或服务器临时错误可继续尝试，最大 100 次；请求被接口拒绝时不会重复发送同一个错误请求。
- 支持通用注释边界块，例如 `<!-- begin_of_名称 -->...<!-- end_of_名称 -->`、`#region 名称...#endregion 名称`、`// begin 名称...// end 名称`。
- 生成预览时点“取消”会真正停止当前生成；关闭 UI 也会先中止后台请求。
- 手机端和电脑端 UI 都做了自适应，按钮保持横向排字并按空间换行。

## 怎么改结构块

不用选中渲染后的块，也不用提前找模板或选择类型。

1. 点聊天输入框旁边的底部魔法棒菜单。
2. 点“回复救急插件”打开 UI。
3. 在 UI 里点“扫描当前聊天块”或直接使用自动扫描结果。
4. 面板会按当前聊天最后一条助手回复的原始文本重新识别结构块。
5. 点你要修的块；需要同时修多个块时，连续点选多个。
6. 在“你的修改要求”里写要改什么。
7. 生成预览，确认后应用。

如果要重新生成整个块，就直接在要求里写“重新生成整个块，但保留原字段顺序和样式”。如果只想改某个值，也直接写“把体力改成 60，其他字段不动”。多选时同一条要求会分别应用到每个选中的块，不再单独提供数字按钮。

## 缺失块和新增块

插件会以当前聊天里结构完整的一轮回复作为“聊天块基线”，记录每个块的识别特征、顺序和大致位置。

- 如果后续某轮回复少生成了某个块，UI 会显示为缺失块，可选中后生成并按基线顺序插回。
- 如果聊天中途合理新增了块，只要这一轮旧块都完整存在，插件会扩展当前聊天的基线，后续也会记住新增块。
- 插入缺失块时会参考前后块位置，不会统一塞到末尾。

## 识别范围

插件不局限于 `<StatusBlock>...</StatusBlock>`，会尝试识别：

- XML/类 XML 成对标签。
- `[块名]...[/块名]`、`【块名】...【/块名】`、`「块名」...「/块名」` 这类成对标记。
- 通用注释边界，例如 `<!-- begin_of_Subtext_think -->...<!-- end_of_Subtext_think -->`、`#region 状态栏...#endregion 状态栏`、`/* begin 面板 */.../* end 面板 */`。
- 带 `class`、`id`、`style`、`data-*`、`aria-*` 的 HTML 美化容器。
- 预设、正则和世界书里的格式线索。

如果原始消息里没有可识别边界，插件不会硬猜，避免误伤正文。

## 依据和防胡编

模型只能依据：

- 旧块或坏块本身。
- 当前回复正文。
- 最近上下文。
- 用户在修改要求里明确给出的信息。
- 预设、正则、世界书提供的格式线索。

预设、正则和世界书只作为格式证据，不作为当前剧情事实。无法确定的字段保持旧值；没有旧值时写“未知”或“无”。

新版默认关闭旧版“无旧块追加”能力。没有当前消息里真实存在的目标块时，不会凭空新建结构块。

## 不粘旧对话

每次打开插件 UI 或点击“扫描当前聊天块”都会重新读取当前聊天最后一条助手回复。

切换或加载聊天时会清空当前块识别状态，避免把上一轮对话的块列表带到新对话里。

## 独立 API

默认使用 SillyTavern 当前配置的模型。如果想让插件单独走另一个接口：

1. 打开 SillyTavern 扩展设置里的“回复救急插件”。
2. 把“生成来源”改成“独立 API”。
3. 填写自定义名称、API URL 和 API Key。
4. 点击“识别可用模型”。
5. 在模型下拉框里选择模型。
6. 点击“保存 API 预设”。

独立 API 使用 OpenAI 兼容接口：`GET /models` 识别模型，`POST /chat/completions` 生成修复结果。请求会交给 SillyTavern 后端转发，避免浏览器直连远端 API 时被 CORS 拦截。API Key 会保存在本地 SillyTavern 扩展设置里，只建议个人电脑使用。

独立 API 不会完整复刻 SillyTavern 的内部发包链路。为了减少接口拒绝请求，插件会在发送前自动整理请求：把约束合并进单条 user 消息、按输入大小压缩或跳过额外预设线索，并根据接口/模型特征收敛实际输出上限。设置里的“自动重试次数”只影响独立 API：填 0 表示关闭，最大可填 100；当接口临时失败、返回空内容、结构块格式不合格，或你写了修改要求但模型返回原样时，会继续尝试直到生成可用预览或达到上限。400/413/422 这类接口拒绝请求不会重复发送，需要检查 API URL、模型名、Key 权限或该接口的上下文/输出限制。

## 手机/电脑适配

电脑端：插件 UI 左侧是块列表，右侧是原文、修改要求和预览。

手机端：自动改为单列，块列表在上，编辑区在下，按钮横向排字并按空间换行。
