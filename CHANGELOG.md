# 更新说明

## v0.1.19

这是一次从 GitHub 旧版 `v0.1.11` 到当前本地新版的整合更新，重点是让“回复救急插件”从单块修复工具升级为可处理多块、缺失块和独立 API 的完整救急面板。

### 新增

- 新增多选结构块修复：在“已识别块”里连续点选多个块后，可一次生成预览，应用时会分别替换每个原始块。
- 新增缺失块识别：插件会记录当前聊天结构完整时的块顺序和位置；后续某轮回复少生成块时，会在 UI 中显示为“缺失”，可选中后重新生成。
- 新增缺失块按位置插回：缺失块应用时会根据聊天块基线和前后块位置插入，不再只能追加到末尾。
- 新增中途新增块处理：如果某轮回复完整包含旧块并额外新增结构块，插件会扩展当前聊天的块基线，后续继续按新版块结构识别。
- 新增删除选中块：可删除当前回复里真实存在的选中块；缺失占位会自动跳过，避免误删。
- 新增生成来源切换：可选择“当前酒馆模型”或“独立 API”。
- 新增独立 API 预设：支持保存多个 API 预设，每个预设包含自定义名称、API URL、API Key、模型、温度和最大输出长度。
- 新增自动识别模型：填写独立 API 的 URL 和 Key 后，可点击“识别可用模型”，插件会调用 OpenAI 兼容 `GET /models` 并在下拉框中列出模型。
- 新增独立 API 生成：选择独立 API 后，修复生成会调用 OpenAI 兼容 `POST /chat/completions`。
- 新增大上下文配置：上下文字符数上限提升到 `100000`。
- 新增大输出配置：修复输出长度上限提升到 `20000`。

### 改进

- 块识别不局限于 `<StatusBlock>...</StatusBlock>`，会继续尝试识别 XML/类 XML、方括号/中文括号成对块、HTML 美化容器、预设/正则/世界书中的格式线索。
- 多选块生成时，每个块都有独立边界标记，减少模型把多个块混在一起的概率。
- 缺失块生成时，聊天基线样例只作为格式和旧字段参考；字段值仍要求来自当前回复正文、最近上下文和用户明确要求。
- 打开 UI 时不再默认选中第一个块，避免用户误以为已经选择了修复目标。
- 手机端块列表、扫描按钮、底部操作按钮进一步自适应，按钮文字保持横向显示并按空间换行。
- 设置面板新增每项功能解释，减少用户猜按钮用途的成本。
- “扫描当前聊天块”保留在主 UI 中，方便随时重新读取当前最后一条助手回复。

### 修复

- 修复某些 HTML/嵌套代码块会被拆成多个错误识别块的问题。
- 修复格式出错后只能反复识别坏块、无法按正确块结构修复的问题。
- 修复某些块明明存在却被标记为缺失的问题，增加基于标记、位置和子容器覆盖的兜底匹配。
- 修复生成预览后无法正确应用到当前正文的问题。
- 修复手机端“已识别块”内容不可见、扫描按钮和修复按钮不好点的问题。
- 修复浏览器缩放后 UI 宽度和按钮布局不跟随页面自适应的问题。

### 使用方式

1. 选中正文片段时，点消息旁的小魔法棒，只修改选中的正文。
2. 要修状态栏、选项栏、生理系统或其他结构块时，点底部魔法棒菜单里的“回复救急插件”打开 UI。
3. 在“已识别块”里选择要修的块；需要同时修多个块就连续点选多个。
4. 在“你的修改要求”里写清楚要改什么。
5. 点击“生成修复预览”，确认后再点击“应用到当前回复”。
6. 如需独立 API，在扩展设置中把“生成来源”切到“独立 API”，填写 URL 和 Key，识别模型后保存预设。

### 注意事项

- 独立 API 需要兼容 OpenAI 的 `/models` 和 `/chat/completions` 接口。
- API Key 会保存在本地 SillyTavern 扩展设置里，只建议个人电脑使用。
- 如果接口不允许浏览器跨域请求，插件会提示无法连接或 CORS 问题；这种情况需要换支持跨域的接口，或后续增加本地代理。
- 插件仍坚持“先预览，后应用”，不会自动改写聊天内容。
